# Base command for work
git init - инициализация проекта.
git add . - добавления всех файлов/а в коммит.
git commit '' - фиксация изменений.
git push - закинуть коммиты на удалённый репозиторий.
git pull - забрать изменения из репозитория.

# Other
git clone - клонировать репозиторий ( можно пропустить инициализацию )
git branch - узнать какие ветки существуют в репозитории.
git checkout - помогает для перемещения между ветками. (switch обновлённая версия)
git merge - слить две ветки. (надо находиться через checkout или switch на одной ветки, а в merge писать другую)
git rebase - слияние веток. (но делает это чище по сравнению с merge) Больше для личных веток не для совместной работой над одной.
git fetch - просмотреть изменения, не изменяя свои файлы.
git diff - посмотреть различия в ветке.
git revert <hash> - изменит commit перевернув его, но сохранит целостность.

# Information
git log --oneline --graph - посмотреть визуализацию веток. 
git status - посмотреть состояние ветки/файлов.


# Questions
- Как перенести коммиты из одной ветки в другую? 
Перейти через checkout на ветку в которой нам нужно сделать слияние через git merge <ветка> (самый простой способ) может так же лишний коммит слияния создать!
Либо через cherry-pick с выборкой нужных коммитов (могут быть конфликты!)

- Что такое merge conflict и каковы причины?
а. Когда, git не понимает как слить 2 ветки т.к. они противоречат друг другу.
б. Разные значения переменной в одном файле при изменение разными разработчиками. 
   Один разработчик удалил файл, другой его изменил ( то есть не удалял ).
(Чаще используйте git pull ребята) РЕКОМЕНДАЦИЯ!

- git pull из под капота как работает? (git pull origin main под капотом)
а. git fetch origin main - забираем без изменений
б. git merge origin/main - вливаем в текущую ветку
(origin/main - это локальная директория!)
